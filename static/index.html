📚 Data Structures Documentation

1. Collection Data Structures

  • Airlines: std::unordered_map<int, Airline> for O(1) lookup by OpenFlights ID, plus std::unordered_map<std::string, int> mapping IATA codes to IDs.
  • Airports: Same dual-map structure as Airlines — std::unordered_map<int, Airport> and std::unordered_map<std::string, int> for IATA indexing.
  • Routes: std::vector<Route> as the primary collection, with three std::unordered_map<int, std::vector<size_t>> indices mapping airline IDs, source airport IDs, and destination airport IDs to vectors of route indices.

2. Route Linkage

Routes reference Airlines and Airports via their numeric OpenFlights IDs (integers). Each Route stores airlineId, sourceAirportId, and destAirportId. The route index maps provide O(1) access to all routes associated with a given airline or airport.

3. Airline Report Algorithm

Given airline IATA code “XX”:

  1. Look up the airline ID via _airlineByIata["XX"] — O(1).
  2. Retrieve all route indices from _routesByAirlineId[airlineId] — O(1).
  3. For each route, increment a counter in a std::map<int, int> for both source and destination airport IDs — O(R) where R = number of routes for this airline.
  4. Sort the airport-count pairs by count descending — O(A log A) where A = unique airports.
  5. For each entry, look up the Airport entity and assemble the JSON response.

4. Scalability Analysis

  • 1000× record size: Switch from storing full structs to using std::shared_ptr with memory-mapped backing store; add LRU cache for hot entities.
  • 1000× entity count: Replace unordered_map with a B-tree or LSM-tree backed store (e.g., RocksDB); add pagination to all reports.
  • 1000× routes: Pre-compute materialized views for common reports; use adjacency lists with compressed sparse row format instead of flat vectors.
  • Ratio change to 1:100 writes: Replace std::vector-based route storage with a balanced BST (std::set) or skip list for O(log n) insert/delete; use write-ahead logging; replace full index rebuilds with incremental index updates.
